# 使用示例：从广告视频到完整游戏设计

本文档演示**参考路径**——从一段游戏广告视频出发，将其转化为完整的游戏设计方案。你不需要描述原创创意，只需描述你在视频中看到了什么，AI 会反向解析玩法并扩展为完整产品。

> **这正是原始 `game-implement-process` 工作流的设计初衷：** 将一段 30 秒的超休闲广告转化为"可玩一周"的完整游戏方案。

---

## 前置条件

与[从零开始的示例](example-walkthrough.zh-CN.md)相同——安装 Game Forge，在 Claude Code 或 OpenCode 中打开你的项目。

---

## 参考素材：一段超休闲广告视频

假设你在刷 Instagram 时看到了这样一条 30 秒广告：

> **广告内容：**
> 玩家在屏幕上画一条线来引导一个小球通过障碍。
> 小球沿着画出的路径滚动，收集金币并避开尖刺。
> 小球到达终点时，金币飞溅爆开，发出令人满足的"叮"声。
> 然后出现"第2关"的画面，障碍布局更复杂。
> 整个过程看起来简单、单手操作、莫名解压。

你想基于这个创意来做一款游戏，但要扩展成完整产品。

## 阶段 1：从参考素材创建策划总纲 (`/gf-concept`)

### 会话 1：描述视频

```
/gf-concept
```

因为你选择了参考路径，AI 的开场不同：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 GAME FORGE ► 策划总纲 — 第 1/5 次会话
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

你选择从参考素材开始。请描述你看到的内容——
一段广告、你玩过的游戏、竞品的产品。尽量详细：
屏幕上发生了什么、玩家做了什么、哪里感觉好或不好。
```

**你的输入：**
```
我在 Instagram 上看到一条 30 秒广告。以下是我观察到的：

画面：干净、极简的美术风格。明亮的颜色配上白色/马卡龙色背景。
小球是荧光色的，滚动时留下发光轨迹。

玩法：玩家用手指在屏幕上画一条线/路径。然后一个小球从起点
沿着你画的路径滚动。路径需要越过空隙、绕过尖刺障碍、穿过
金币群。小球完全沿着你画的路径走——基于物理的滚动。

爽感：小球碰到金币时，金币爆开产生粒子效果和"叮"的音效。
到达终点时，大量金币爆炸飞出，屏幕震动。非常令人满足的
声画反馈。

失败：如果小球撞到尖刺或掉出路径，它会"啪"地碎裂，
然后重新开始关卡。快速重开，没有惩罚。

关卡结构：看起来是短关卡（每关 10-15 秒）。完成一关后，
出现"第2关"画面，障碍布局更难。

商业化：我看到通关后有一个"x2 金币"按钮带播放图标
（激励视频广告）。游戏过程中底部还有横幅广告。
```

**AI 分析参考素材并提出第 1 章方案：**

```
很好的参考素材。我的分析：

核心机制：画线引导物理解谜
类型：超休闲，广告变现，单次会话
卖点："画路径 → 看小球滚动 → 爽感结果"
留存驱动：快速关卡 + 优化冲动（"画更好的路径"）

在我起草第 1 章之前，想了解你的目标：

1. 你想在广告的基础上扩展多远？
   我建议：保留画线引导的核心，但加入元系统层
   （解锁球皮肤、主题章节、每日挑战）来延长
   从"玩 5 分钟"到"玩一周"的生命周期。
   同意，还是更接近超休闲原版？

2. 目标用户——广告瞄准的是超轻度用户（任何有手机的人，
   零游戏经验即可）。保持这个定位，还是偏向
   需要更多深度的轻核玩家？

3. 商业化优先级——广告展示了激励视频 + 横幅。
   保持这个组合，还是侧重某一种？我建议：
   主力：激励视频（2倍金币、续命、道具）
   辅助：插屏（每 3-4 关之间）
   补充：去广告内购（¥18）
```

**你回复：**
```
1. 同意，做元系统层扩展，解锁和章节都要。
2. 保持超轻度定位，但加足够的深度支撑一周游玩。
3. 同意你的商业化建议。
```

**AI 生成第 1 章，预填了从参考视频推导的内容：**

```markdown
# 第 1 章：目标用户与使用场景

## 1.1 目标用户画像

**R_1_01** | 配置类型：user_profile
主用户：18-45 岁，不限性别，超轻度手游用户。
零游戏经验即可上手。画线操作直觉——
任何会用触摸屏的人都能玩。

> **来自参考：** 广告的视觉语言（明亮、干净、极简）
> 瞄准最广泛的受众。无文字密集UI，无复杂操控，
> 无特定游戏类型知识门槛。

**R_1_02** | 配置类型：session_params
单次游玩时长：2-10 分钟（每次 3-8 关）。
即时启动——无加载画面、无登录、无教学菜单。
打开应用 3 秒内玩家就在第一关画线。

> **来自参考：** 广告展示了即时进入的游戏画面——
> 无菜单、无剧情铺垫、无角色选择。第一次触碰就是游戏本身。
```

注意每个章节都包含 **"来自参考"** 标注，解释哪些内容是从视频描述推导的。用户可以确认或覆盖这些推导。

### 会话 2-5：超越广告的扩展

这是 AI 发挥价值的地方——将 30 秒的广告概念扩展为完整游戏：

**第 3 章（核心玩法）— AI 扩展画线机制：**

```
第 3 章：核心玩法设计

广告展示了基础的画线和滚动。为了支撑一周的游玩，
我建议从 3 个维度扩展机制：

1. 画线工具（可解锁）：
   - 自由画线（手绘） — 一开始就有
   - 直线工具 — 第 10 关解锁
   - 曲线工具（贝塞尔） — 第 20 关解锁
   - 橡皮擦（局部重画） — 第 30 关解锁

2. 障碍物多样性（按章节）：
   - 第 1 章：静态尖刺、空隙
   - 第 2 章：移动平台、计时门
   - 第 3 章：重力区域（小球上坡/下坡）
   - 第 4 章：传送门（小球在两点间传送）
   - 第 5 章：多球分裂

3. 评分深度：
   - 路径越短 = 星级越高（效率奖励）
   - 沿途金币收集
   - 时间奖励（画线 + 滚动在时限内完成）
   - "完美路径"奖励（无重画、不碰墙）

> **来自参考：** 广告只展示了基础画线 + 金币 + 尖刺。
> 这些扩展保持了"画线然后观看"的满足感，同时增加了
> 足够的变化支撑 100+ 个关卡。

同意这些扩展方向，还是要调整？
```

### 质量门禁

质量门禁在所有章节完成后运行，参考路径有一个额外检查：

```
参考一致性检查：
  ✓ 所有参考观察点都已在章节中体现
  ✓ 超出参考的扩展内容已明确标注
  ✓ 核心机制保持不变（画线引导）
  ⚠ 参考视频中游戏过程有横幅广告——第 09 章改为
    放在关卡之间。是否有意为之？（是/否）
```

## 阶段 2-4：流程相同，但感知参考素材

后续阶段与[从零开始的示例](example-walkthrough.zh-CN.md)流程一致，关键区别是：AI 在做设计决策时会引用原始视频观察。

**系统设计示例：**
```markdown
# SYS-DRAW — 画线引擎

## 4.5 核心规则

RULE-DRAW-001：玩家通过触摸拖拽在屏幕上画路径
> 来自参考：连续单笔画，不是点击放置路径点

RULE-DRAW-002：路径有最小线段长度（防止画点）
> 来自参考：广告展示的是平滑曲线，不是锯齿短线段

RULE-DRAW-003：路径不能自交叉
> 设计决策：参考中未显示，但防止利用交叉走捷径

RULE-DRAW-004：路径与障碍物碰撞使该段无效
> 设计决策：参考中未展示——防止画线穿墙
```

**数据建表示例：**
```markdown
### level_config（关卡配置表）

| 字段 | 类型 | 默认值 | 必填 | 含义 | 示例 |
|------|------|--------|------|------|------|
| level_id | int | - | 是 | 关卡编号 | 1 |
| chapter_id | int | 1 | 是 | 所属章节 | 1 |
| start_x | float | - | 是 | 小球起点 X | 0.1 |
| start_y | float | - | 是 | 小球起点 Y | 0.9 |
| end_x | float | - | 是 | 终点 X | 0.9 |
| end_y | float | - | 是 | 终点 Y | 0.1 |
| obstacles | string | "" | 否 | 障碍物 JSON 数组 | [{"type":"spike","x":0.5}] |
```

## 参考路径 vs 从零开始的关键区别

| 方面 | 从零开始 | 从参考视频 |
|------|---------|-----------|
| 第一个问题 | "跟我说说你的游戏创意" | "描述你看到了什么" |
| 章节预填 | 空白——AI 全部询问 | AI 从参考推导后预填，用户确认 |
| 标注 | 无 | 文中标注"来自参考" |
| 扩展提示 | "需要什么功能？" | "想在参考基础上扩展多远？" |
| 质量门禁 | 标准检查 | + 参考一致性检查 |
| 系统设计 | 纯共创 | 参考观察作为规则来源注释 |
| 可追溯性 | 策划规则 → 系统 | 参考观察 → 策划规则 → 系统 |

## 什么时候用参考路径

- 看到一条游戏广告，想做类似但更好的产品
- 玩了竞品的游戏，想做差异化
- 客户或老板给了一段参考视频
- 想"借鉴+改良"已有创意
- 想验证观察到的机制能否支撑完整产品

## 描述视频参考的技巧

你提供的细节越多，AI 预填章节的效果越好：

**好的描述（AI 能预填第 1-3 章的大部分内容）：**
```
玩家点击屏幕跳跃。双击二段跳。不同高度有平台，
上方漂浮着金币。掉落平台不会死，只是落到下层轨道。
每 10 个平台有一个存档旗帜。像素风美术，有视差滚动
背景。死后看到"看广告恢复存档点"按钮。
```

**差的描述（AI 需要问很多问题）：**
```
是个跳跃游戏，有金币。看着挺好玩挺简单的。
```

参考路径在你能描述以下内容时效果最好：
1. **玩家做什么**（输入方式、核心动作）
2. **屏幕上发生什么**（视觉反馈、成功/失败）
3. **关卡怎么运作**（结构、推进、难度）
4. **怎么赚钱**（广告位置、内购提示）
5. **看起来/听起来怎样**（美术风格、音效反馈）
