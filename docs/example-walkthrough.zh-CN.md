# 使用示例：设计一款三消益智游戏

本文档演示如何使用 Game Forge 从零开始设计一款完整的三消益智游戏。展示流程中每个阶段的实际输入和输出。

> **预计耗时：** 全流程约需 15-20 次会话。本文档展示每个阶段的关键片段。

---

## 前置条件

1. 将 Game Forge 安装到你的项目中：
```bash
git clone https://github.com/citronetic/game-designer.git
cp -r game-designer/.claude/skills/ your-project/.claude/skills/
cp -r game-designer/.claude/agents/ your-project/.claude/agents/
cp -r game-designer/bin/ your-project/bin/
```

2. 在 Claude Code（或 OpenCode）中打开你的项目：
```bash
cd your-project
claude
```

## 阶段 1：策划总纲 (`/gf-concept`)

### 会话 1：创意描述 + 第 1-2 章

```
/gf-concept
```

AI 用一个开放性问题开场：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

跟我说说你的游戏创意。
```

**你的输入：**
```
我想做一个糖果/甜品主题的三消游戏。
特别之处是消除后的糖果会产生连锁组合效果——
比如爆炸可以清除整行、整列或范围内的糖果。
我希望它让人觉得很爽、容易上瘾，目标用户是
通勤或休息时玩手机的休闲玩家。
```

**AI 提出第 1 章（目标用户）的方案：**
```
第 1 章：目标用户与使用场景

我对这一章有初步方案，你来确认：

主用户：25-44 岁女性，休闲手游用户，在通勤/休息时玩，
偏好低压力益智游戏，追求视觉满足感。单次游玩：5-15 分钟。

次用户：16-24 岁年轻人，追求快速多巴胺刺激，
更在意高分和排行榜。

典型场景：
- 早通勤（地铁/公交，10-15 分钟）
- 工作间隙（5-10 分钟，需要快速开始/结束）
- 晚间放松（20-30 分钟，较长会话）

同意这个方向，还是要调整？
```

**你确认**后，AI 生成 `ch01-target-users.md`，包含完整内容（800+ 字）和规则 ID：

```markdown
## 1.1 目标用户画像

**R_1_01** | 配置类型：user_profile
主用户：25-44 岁女性，休闲手游用户……

**R_1_02** | 配置类型：session_params
单次游玩时长：5-15 分钟，必须支持即时开始/结束……
```

然后进入**第 2 章（产品定位）**，问你：

```
第 2 章：产品定位与差异化

在我出稿之前，先确认几个问题：

1. 你的一句话定位？我建议：
   "一款糖果主题三消游戏，爆炸连锁组合将简单的交换
   变成视觉盛宴。"
   同意，还是换个角度？

2. 主要竞品——我想到 Candy Crush、Toon Blast、Cookie Jam。
   还有其他的吗？

3. 我们明确不做什么？建议排除：
   社交功能、实时PvP、复杂剧情。
   同意吗？
```

### 会话 2-6：剩余章节

每次会话处理 2-3 章。AI 的工作方式：
- 简短介绍："第 5 章定义你的关卡和内容管线。"
- 问 3-5 个针对性问题并给出强建议
- 生成完整章节，包含规则 ID 和数据承载类型标记

**第 3 章（核心玩法）的示例：**
```markdown
## 3.2 基础玩法规则

**R_3_01** | 配置类型：关卡配置
棋盘：8×8 格，6 种糖果颜色（可通过配置扩展）。
交换：只能交换相邻糖果，交换必须形成 3 个以上的匹配。
如果没有可用的交换，棋盘自动重排。

**R_3_02** | 配置类型：组合配置
组合规则：
- 4 个一行 → 条纹糖果（清除整行或整列）
- L/T 形 5 个 → 包装糖果（3×3 爆炸）
- 5 个一行 → 彩虹糖（清除所有同色）
- 两个特殊糖果组合 → 超级组合（效果因组合而异）

**R_3_03** | 逻辑实现项：match_engine
匹配检测：每次交换后和每次重力结算后，
扫描棋盘寻找 3+ 连续组。优先处理最大组。
连锁反应：每一步连锁将组合倍率 +1。
```
